## 使用方法

# 主控:

1.在主控机上运行如下命令:
`wget -N --no-check-certificate https://raw.githubusercontent.com/BitsFlowCloud/Looking-Glass/refs/heads/main/install.sh && chmod +x install.sh && bash install.sh`

2.选择 `1 安装主控端` 回车

3.等待安装nginx和php

4.输入安装路径:回车(没有更改的意义)

5.站点基本信息/顶部大标题/页脚文本 随意填!

6.是否启用Cloudflare Turnstile:商家的话建议开启,玩玩的话不要开,选n;

7.设置一个节点注册令牌,这个设置好了要记住!(不用太复杂,就是注册节点用的)

8.Nginx配置-输入IP或者域名,默认按回车就是用IP作为访问入口,你可以输入你的域名,我这里直接回车

到此为止,主控就安装好了!现在可以去看看空无一物的样子啦~

---

# 被控:

1.在被控机上运行如下命令:
`wget -N --no-check-certificate https://raw.githubusercontent.com/BitsFlowCloud/Looking-Glass/refs/heads/main/install.sh && chmod +x install.sh && bash install.sh`

2.选择`2 安装被控端`

3.等待安装nginx和php

4.输入安装路径:回车(没有更改的必要性)

5.设置一个节点秘钥,这是和api.php通信的唯一凭证!可以设置的复杂点,忘了可以通过在主控上运行 `cat /var/www/html/lg/config.php` 查看

6.节点名称:自己填写,然后回车

7.地区代码`为2位英文字母且必须小写` 这样会生成一个国旗(比如us或jp或sg等等)

8.公网IPv4(自动填上了),直接回车

9.公网IPv6(自动填上了),直接回车,接下来会生成一个1G的测速文件

10.自动检测流媒体信息,确保被添加之后就会立马显示

11.自动配置tcp参数(毕竟不开bbr测速没速度)

12.自动配置防火墙(放行iperf3所需端口)

13.输入域名或IP:建议直接IP,默认已填上,直接回车

14.是否上报到主控(一定要选择Y)

15.主控 API地址:比如我的主控是http://47.79.98.178/ ,那么主控地址就是http://47.79.98.178/api.php (需要手动输入下)

16.输入刚才你配置的注册令牌

17.返回主控,节点出现啦!

---

到此为止,主控和被控的配置就全部完成了!
如果你想添加多个节点,在多个被控机依次执行脚本安装被控即可!
